# Architecture Decision Records

This directory captures decisions whose rationale is not obvious from the
code. Each ADR is a short markdown file numbered sequentially:
`NNNN-short-title.md`.

ADRs are append-only — when a decision is superseded, a new ADR records
the new decision and references the old one. Old ADRs aren't edited or
deleted; the historical context is the value.

Each ADR is a distilled single-page rationale for the most-impactful
choices that aren't obvious from reading the code.
